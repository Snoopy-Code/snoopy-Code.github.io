MySQL易混淆知识点

delete和truncate

1 delete删除后，标识从断点开始，truncate删除表示从1开始

2 delte后面可以加筛选条件where，truncate不可以

3 delete效率比truncate低一点

4 在事务中，delete可以回滚，truncate不可以回滚

主键和唯一键的异同点

1 主键和唯一键都可以保证数据不可重复

2 主键至多只有一个，而唯一键可以有多个

3 主键不可以为空，唯一键可以

视图和表

创建视图用view,表是table

视图占用的物理空间很小，只保存了sql逻辑

视图一般只用于查询，而表可以增删改查

MySQL表操作

修改列名

alter table 表名 change colume 原列名 新列名 列类型

修改列类型

alter table 表名 modify colume 列名 新类型

删除列

alter table 表名 drop colume 列名

修改表名

alter table 表名 rename 【to 】 新表名

添加列

alter table 表名 add colume 列名 列类型

删除表

drop table [if exists] 表名

复制表

只复制表的结构

create table 新表名 like 旧表名

复制表的结构和数据

create table 新表名

select * from 旧表名 【where 筛选条件】

如果只要部分表的字段

create table 新表名

select id,name

from 旧表名

where 0          只需要让条件不满足，这样就差不到数据，只显示字段了！

MySQL查询顺序

多条件查询时的顺序

select 查询字段    ⑦

from 表     ①

【连接类型】join     ②

ON 查询条件    ③

where 筛选条件    ④

group by    ⑤

having 筛选条件   ⑥

order by   ⑧

limit    ⑨

当where后面的条件不是标量子查询时，即查询结果为一行一列时，可以使用在from后面添加子查询，作为一个新表，起别名用内连接查询。
或者也可以在where后面使用in关键字查询，in里面的子查询可以列子查询也可以是标量子查询。

WebMvcConfigurerAdapter过时的替换方法

Spring 5.0后，WebMvcConfigurerAdapter被废弃，取代的方法有两种:
① implements WebMvcConfigurer
② extends WebMvcConfigurationSupport
使用第一种方法是实现了一个接口，可以任意实现里面的方法，不会影响到Spring Boot自身的@EnableAutoConfiguration，而使用第二种方法相当于覆盖了@EnableAutoConfiguration里的所有方法，每个方法都需要重写，比如，若不实现方法addResourceHandlers()，则会导致静态资源无法访问。所以推荐使用第一种。

@Controller和@RestController的区别

@Controller配合视图解析器，return返回会跳转到相应的jsp页面
@RestController相当于@ResponseBody和@Controller的合体，它不会跳转到jsp页面，配置的视图解析器失效，返回的就是return里面的内容。一般会会使用在返回时Json数据.
