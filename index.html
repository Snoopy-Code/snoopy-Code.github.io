mysql索引今日小总结

索引就是能够快速查找的排好序的数据结构

可以使用explain+SQL语句对于语句进行优化，没有处理的SQL语句，他的Type都是ALL，即

全变搜索，这样当数据很多的时候就会很慢。根据SQL语句后面的条件适当的进行的索引建立

一般建立比较多的索引是复合索引，在查询语句时尽量少用*，可以使用覆盖索引，尽可能的精确

到指定字段。

对于索引失效问题，例如建立复合索引 create index 索引名 on 表名(字段…)

例如index(a,b,c）

首先呢，全职匹配是最好的，select a,b,c from …..这样的查询符合覆盖索引，索引也不会失效

其次就是最左前缀法则，你所查询的条件必须是以a为开头，尽量不要跳中间，就是select * from a

where a = ‘a1’and c=’c1’,where后面必须以a为第一个条件，否则索引失效，中间不要跳###

这句SQL跳了b，所以后面的索引会失效，只适用了部分索引

第三，不要在索引上做任何计算，函数，以及隐式的类型转换，否则会失效。

第四呢，就是字段使用范围条件，比例> ,< ,between and ,in 等等，会使该字段后面的索引失效

尽可能的少使用select *，多使用覆盖索引

SQL语句在使用!= , <>,is null或is not null 都会使索引失效

like通配符只可以放在字段右边，就是’aa%’,不可以’%aa%’或者’%a’，因为前面说了最左前缀法则

开头使用了%,不知道到底里面包含几个字段，所以索引失效

varchar类型的数据不可以不加单引号括起来，否则会失效，虽然看着很低级，但是如果name字段是

varchar类型，假如name是2000，那么会很容易手误写成name =2000…

尽可能少用or，用它来连接时会使索引失效，但并不是一点都不能用

其次就是在用explain优化sql时，查看执行结果注意个字段的含义，尤其id type select_type key

2020-03-18

Docker容器中的mysql插入数据无法使用中文

进入容器输入命令查看locale查看容器内支持的字符集

会发现是POSIX,这个字符集是不支持中文的，需要将其改成utf8

使用命令 docker exec -it 容器名称 env LANG=C.UTF-8 /bin/bash

注意：这个是临时的，关闭容器后会重置字符集。

所以每次进入容器都需要设置，想要永久，需要在dockerfile中设置。

MySQL易混淆知识点

delete和truncate

1 delete删除后，标识从断点开始，truncate删除表示从1开始

2 delte后面可以加筛选条件where，truncate不可以

3 delete效率比truncate低一点

4 在事务中，delete可以回滚，truncate不可以回滚

主键和唯一键的异同点

1 主键和唯一键都可以保证数据不可重复

2 主键至多只有一个，而唯一键可以有多个

3 主键不可以为空，唯一键可以

视图和表

创建视图用view,表是table

视图占用的物理空间很小，只保存了sql逻辑

视图一般只用于查询，而表可以增删改查

MySQL表操作

修改列名

alter table 表名 change colume 原列名 新列名 列类型

修改列类型

alter table 表名 modify colume 列名 新类型

删除列

alter table 表名 drop colume 列名

修改表名

alter table 表名 rename 【to 】 新表名

添加列

alter table 表名 add colume 列名 列类型

删除表

drop table [if exists] 表名

复制表

只复制表的结构

create table 新表名 like 旧表名

复制表的结构和数据

create table 新表名

select * from 旧表名 【where 筛选条件】

如果只要部分表的字段

create table 新表名

select id,name

from 旧表名

where 0          只需要让条件不满足，这样就差不到数据，只显示字段了！

MySQL查询顺序

多条件查询时的顺序

select 查询字段    ⑦

from 表     ①

【连接类型】join     ②

ON 查询条件    ③

where 筛选条件    ④

group by    ⑤

having 筛选条件   ⑥

order by   ⑧

limit    ⑨

当where后面的条件不是标量子查询时，即查询结果为一行一列时，可以使用在from后面添加子查询，作为一个新表，起别名用内连接查询。
或者也可以在where后面使用in关键字查询，in里面的子查询可以列子查询也可以是标量子查询。
